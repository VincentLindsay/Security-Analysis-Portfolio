# Configuring Suricata
* Now that both items have been downloaded, I installed Suricata to the VM
<img width="488" height="382" alt="image" src="https://github.com/user-attachments/assets/927eb1df-9431-4c87-9e02-c9426dfac425" />

* Suricata was successfully installed to the VM, and now we can configure Suricata to suit our needs
* I edited the **suricata.yaml** file to include the network adapter of the Windows VM
<img width="890" height="87" alt="image" src="https://github.com/user-attachments/assets/2ab4f442-55d2-433a-ad24-9033a058798b" />

* The section to be edited is the **af-packet** section,which i changed the interface with **Ethernet**
* This was found by using the following command:
```Bash
ipconfig /all
```
<img width="757" height="86" alt="image" src="https://github.com/user-attachments/assets/7408147b-5f59-45e9-968c-eddc46d45028" />

* Since Suricata also uses rule files to detect threats, I used the latest ruleset from emerging [threats](https://rules.emergingthreats.net/open/)
<img width="965" height="331" alt="image" src="https://github.com/user-attachments/assets/4b7aa712-37ac-4744-b50d-6137d9846c0d" />

* I selected the version that is most recent for Suricata
* Select the emerging-all.rules [page](https://rules.emergingthreats.net/open/suricata-7.0.3/emerging-all.rules)
* Once downloaded, I added the rules file to the **rule-files** section of the **suricata.yaml** file
<img width="307" height="45" alt="image" src="https://github.com/user-attachments/assets/496da5fa-4bf4-4b28-ab51-bd85b966d183" />

* Ensure you also move the emerging-all.rules file to the **rules** folder for suricata, which can found in this path:
```Bash
C:\Program Files\Suricata\rules
```
* In the **emerging-all.rules** file, I removed the hashtags from the alerts to enable them
  <img width="616" height="351" alt="image" src="https://github.com/user-attachments/assets/7ee5e42b-2e6d-4426-91cb-94aa394871c0" />


* Next, I executed suricata to verify its ability to operate in the VM using this command:
```Bash
suricata.exe -c suricata.yaml -i Ethernet
```
* This executes Suricata with the correct configuration file, and network interface 
<img width="831" height="107" alt="image" src="https://github.com/user-attachments/assets/f18c0e3c-9fb6-4537-adeb-156cd8ab94e7" />

# Customizing Suricata
* This section involves the creation of a custom log folder for which suricata will send all of the alerts to. I will also re configure Suricata to run as a Windows service so that it will automatically run.

* I created a separate log file for testing purposes
<img width="851" height="86" alt="image" src="https://github.com/user-attachments/assets/6e39fcd3-5b3c-49a6-8f36-e303c267db5d" />

* With the following command, I can run suricata as a service using the **suricata.yaml** configuration settings, using Windows VM IP address:
```Bash
suricata.exe -c suricata.yaml -i 192.168.190.132 -l Lab --service-install
```
* By changing the service configuration to automatic, Suricata will automatically run, instead of having to use the terminal everytime.
